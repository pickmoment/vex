# VEX — Visual EXplorer

> "파일을 보는 새로운 방식 — 터미널 안의 뷰어"

VEX는 Rust로 구현된 TUI(Terminal User Interface) 파일 매니저입니다.  
파일 **보기(View)** 경험에 집중하고, 하단 힌트 바와 마우스 지원으로 직관적인 조작을 제공합니다.

## 설치

**사전 요구사항:** Rust 1.75+ (`rustup` 권장)

```bash
git clone https://github.com/example/vex
cd vex

# 릴리즈 빌드
cargo build --release
./target/release/vex

# 또는 PATH에 설치
cargo install --path .
```

## 기능

### 마크다운 렌더링

`pulldown-cmark` 기반으로 파싱하여 TUI 위젯으로 렌더링합니다.

지원 요소: H1~H6 제목, **굵기**, *기울임*, `인라인 코드`, 코드 블록, 테이블, 인용구, 체크리스트, 수평선

**Frontmatter**: 문서 앞의 `---` YAML 블록을 파싱해 속성/값 테이블로 표시합니다. 리스트 값은 쉼표로 합쳐 한 셀에 표시됩니다.

**줄바꿈 모드(`w`)에서 표**: 테이블이 화면 너비에 맞게 컬럼 너비를 축소하고, 셀 내용이 길면 테두리를 유지한 채 셀 내부에서 줄바꿈됩니다.

### 신택스 하이라이팅

`syntect` 기반으로 170개 이상의 언어를 지원합니다.

Rust, Python, JavaScript/TypeScript, Go, C/C++, Java, Shell, TOML, YAML, JSON, HTML, CSS 등

### 이미지 미리보기

터미널 환경을 자동 감지하여 최적의 프로토콜로 렌더링합니다.

| 프로토콜 | 지원 터미널 |
|----------|------------|
| Kitty | kitty |
| iTerm2 | iTerm2 (macOS) |
| Sixel | WezTerm, xterm 등 |
| Braille | 모든 터미널 (폴백) |

지원 포맷: PNG, JPEG, GIF, WEBP, BMP

### CSV 테이블 뷰어

CSV/TSV 파일을 컬럼 정렬된 인터랙티브 테이블로 표시합니다.

### 파일 검색

`/` 키로 현재 디렉토리에서 파일명 실시간 필터링, 매칭 부분 강조 표시.

## 파일 관리

파일 탐색 중 `m` 키로 파일 관리 메뉴를 엽니다.

```
┌─ 파일 관리: README.md ──────────────────────────────────────────┐
│                                                                  │
│ ▶  복사       대상 경로로 복사              [c]                   │
│    이동       대상 경로로 이동              [v]                   │
│    이름 변경  현재 디렉토리에서 이름 변경    [r]                   │
│    삭제       파일/디렉토리 삭제             [d]                   │
│    새 폴더    현재 디렉토리에 새 폴더 생성   [n]                   │
│                                                                  │
│  ↑↓:이동  Enter:선택  Esc/q:취소                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 메뉴 단축키

| 키 | 동작 |
|----|------|
| `↑` / `↓` | 항목 이동 |
| `Enter` | 선택한 작업 실행 |
| `c` | 복사로 바로 이동 |
| `v` | 이동으로 바로 이동 |
| `r` | 이름 변경으로 바로 이동 |
| `d` | 삭제로 바로 이동 |
| `n` | 새 폴더 생성으로 바로 이동 |
| `Esc` / `q` | 메뉴 닫기 |

### 복사 / 이동

대상 경로 입력 창이 열립니다. 현재 디렉토리가 기본값으로 채워집니다.
`Tab` 키로 경로 클립보드에서 미리 저장한 경로를 선택할 수 있습니다.

```
┌─ 복사: README.md ──────────────────────────────────────────────┐
│                                                                 │
│  원본: README.md                                                │
│                                                                 │
│  대상 경로: /home/user/backup/█                                 │
│                                                                 │
│  Enter:확인  Tab:경로목록  Esc:메뉴로                            │
└─────────────────────────────────────────────────────────────────┘
```

대상 경로가 이미 존재하면 덮어쓰기 여부를 확인합니다 (`y` 확인 / `n` 취소).

### 이름 변경

현재 파일명이 입력창에 미리 채워집니다.

```
┌─ 이름 변경: README.md ─────────────────────────────────────────┐
│                                                                 │
│  원본: README.md                                                │
│                                                                 │
│  새 이름: README_backup.md█                                     │
│                                                                 │
│  Enter:확인  Esc:메뉴로                                          │
└─────────────────────────────────────────────────────────────────┘
```

### 삭제

`y` 키로 삭제를 확인합니다. 디렉토리는 하위 내용 포함 전체 삭제됩니다.

```
┌─ 삭제 확인 ─────────────────────────────────────────────────────┐
│                                                                  │
│  ⚠  정말 삭제하시겠습니까?                                        │
│                                                                  │
│  README.md                                                       │
│                                                                  │
│  y:삭제  n/Esc:취소                                              │
└──────────────────────────────────────────────────────────────────┘
```

작업 완료 후 파일 목록과 Git 상태가 자동으로 갱신됩니다. 성공·실패 결과는 하단 상태 바에 토스트 메시지로 표시되며 3~5초 후 자동으로 사라집니다.

## Git 관리 모드

파일 탐색 중 `g` 키로 진입합니다. Git 저장소가 아닌 경우 안내 메시지를 표시합니다.

### 파일 상태 관리

```
┌─ Git: main ↑2↓0  origin/main  스테이징 2  변경 3 ─────────────┐
├─ 스테이징 (2) ─────────┬─ diff: src/app.rs ────────────────────┤
│ ▶  M  src/app.rs      │ diff --git a/src/app.rs b/src/app.rs │
│    A  src/git/mod.rs  │ @@ -10,5 +10,20 @@                   │
├─ 워킹 트리 (3) ─────── │  pub struct App {                    │
│    M  Cargo.toml      │ +    pub git_diff_wrap: bool,         │
│    M  README.md       │ -    old_field: usize,                │
│    ?? src/new_file.rs │                                       │
├───────────────────────┴───────────────────────────────────────┤
│ a:스테이지  u:언스테이지  c:커밋  f:전체화면  L:로그  q:닫기        │
└────────────────────────────────────────────────────────────────┘
```

파일을 선택하면 오른쪽 diff 패널에 내용이 즉시 표시됩니다.

헤더에는 **트래킹 브랜치명**과 **↑N↓M** 형식의 ahead/behind 카운트가 표시됩니다. diff 힌트 패널 하단에는 원격 URL, 트래킹 브랜치, push/pull 필요 여부가 표시됩니다.

| 키 | 동작 |
|----|------|
| `↑` / `↓` | 파일 이동 (diff 자동 갱신) |
| `Tab` | 스테이징 ↔ 워킹 트리 전환 |
| `a` | 선택 파일 스테이지 |
| `u` | 선택 파일 언스테이지 |
| `X` | 선택 파일 변경 되돌리기 (워킹 트리, 확인 필요) |
| `c` | 커밋 메시지 입력 후 커밋 |
| `b` | 브랜치 패널 열기 |
| `p` | pull (비동기) |
| `P` | push (비동기, `--set-upstream`) |
| `F` | fetch --all --prune (비동기) |
| `!` | force push 확인 후 실행 (`--force-with-lease`) |
| `r` | 상태 새로고침 |
| `L` | 커밋 로그 패널 열기/닫기 |
| `q` | 파일 탐색으로 돌아가기 |

### 커밋 로그 탐색

`L` 키로 3패널 로그 뷰로 전환됩니다.

```
┌─ 커밋 로그 ────────────┬─ 변경 파일 ──────┬─ diff ──────────────┐
│ ▶ abc1234 Fix bug     │ ▶  M src/app.rs  │ diff --git a/...   │
│   def5678 Add feature │    A src/new.rs  │ @@ -5,3 +5,7 @@    │
│   ghi9012 Initial     │    D old_file.rs │ -old line          │
│                       │                  │ +new line          │
├───────────────────────┴──────────────────┴────────────────────┤
│ ↑↓:커밋이동  →/Enter:파일목록  ↑↓:파일이동  f:전체화면  q:닫기     │
└────────────────────────────────────────────────────────────────┘
```

| 단계 | 키 | 동작 |
|------|----|------|
| 로그 열기 | `L` | 커밋 로그 패널 표시 |
| 커밋 목록 | `→` / `l` | 커밋 목록 포커스 진입 |
| | `↑` / `↓` | 커밋 이동 (변경 파일 자동 로드) |
| | `→` / `Enter` | 변경 파일 목록 포커스 진입 |
| 변경 파일 | `↑` / `↓` | 파일 이동 (diff 자동 로드) |
| | `←` / `Esc` | 커밋 목록으로 돌아가기 |

### diff 뷰어 조작

파일 diff 및 커밋 diff 모두 동일한 조작키를 사용합니다.

| 키 | 동작 |
|----|------|
| `PageUp` / `PageDown` | 수직 스크롤 |
| `[` / `]` | 수평 스크롤 (±4칸) |
| `w` | 줄바꿈 ON/OFF |
| `f` | diff 전체화면 |
| `↑`/`↓`, `←`/`→`, `PageUp`/`PageDown`/`Space` | 전체화면 내 스크롤 |
| `Esc` / `f` | 전체화면 닫기 |

### 브랜치 관리

파일 상태 화면에서 `b` 키로 브랜치 패널을 엽니다.

```
┌─ 브랜치 (로컬 3 · 원격 2) ──────────────────────────────────────┐
│  abc1234  ▶ * main                      Initial commit        │
│  def5678    feature/login               Add login page        │
│  ghi9012    bugfix/typo                 Fix typo              │
│  ─────────────────────────────────────────────────────────    │
│  abc1234  ◇   origin/main              Initial commit        │
│  def5678  ◇   origin/feature/login     Add login page        │
└──────────────────────────────────────────────────────────────┘
```

`▶ *` 는 현재 브랜치, `◇` 는 원격 브랜치를 나타냅니다.

| 키 | 동작 |
|----|------|
| `↑` / `↓` | 브랜치 이동 |
| `Enter` | 선택 브랜치로 전환 (`git switch`) |
| `n` | 새 브랜치 생성 후 전환 (`git switch -c`) |
| `d` | 브랜치 삭제 (병합된 브랜치만, 확인 필요) |
| `D` | 브랜치 강제 삭제 (미머지 포함, 확인 필요) |
| `r` | 브랜치 목록 새로고침 |
| `b` / `Esc` | 패널 닫기 |

### 원격 동기화

push/pull/fetch는 비동기로 실행되며 진행 중 모달이 표시됩니다.

```
┌─ push 진행 중 ──────────────────────────────────────────────────┐
│                                                                  │
│  ⠹  git push --set-upstream origin main                         │
│                                                                  │
│  [Esc] 취소                                                      │
└──────────────────────────────────────────────────────────────────┘
```

| 키 | 동작 |
|----|------|
| `p` | `git pull` (현재 브랜치) |
| `P` | `git push --set-upstream origin <branch>` |
| `F` | `git fetch --all --prune` |
| `!` | force push 확인 후 `--force-with-lease` 실행 |
| `Esc` | 진행 중인 비동기 작업 취소 |

> **참고**: detached HEAD 상태에서는 push/pull을 차단합니다. force push는 `--force-with-lease` 를 사용해 원격 커밋이 예상치 않게 변경된 경우를 감지합니다.

## 설정

설정 파일 위치:

| OS | 경로 |
|----|------|
| macOS | `~/Library/Application Support/vex/config.toml` |

```toml
args = []
terminal = true

[[openers]]
name = "VS Code"
command = "code"
args = []
terminal = false

# 즐겨찾기
bookmarks = [
  "/home/user/projects",
  "/home/user/Documents",
]
```
